How It Works:

1
Greedy Matching: Comparing detections to tracked objects. If IoU > 0.3, they are the same fruit.
2
Object Permanence: The simulation in the Flutter diagram (right) shows our "Sticky ID" logic. Even when fruits cross paths, IDs remain unique.
3
Temporal Smoothing: Using a weighted average (0.7*old + 0.3*new) to ensure smooth bounding box movement.
4
Counting Logic: Global count increments only on new ID assignment, preventing double-counting.
